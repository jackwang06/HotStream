# PRD: 文旅热点借势营销

## Executive Summary

HotStream 当前不应被定义为单纯的“热点抓取网站”，而应定位为面向文旅甲方的“热点借势营销辅助工具”。系统需要从多平台发现热点，判断热点是否适合与伊吾、前山牧场、四季牧歌民俗风情园及周边景点产生关联，并辅助生成可发布的宣传文案、图片素材和编辑内容。

本阶段需求更新重点：

1. 将产品目标从“抓热点生成文案”调整为“蹭合适的文旅热点宣传甲方景点”。
2. 引入更适合文旅传播的数据来源，尤其是抖音和微博。
3. 明确抖音热门视频的分析方法，用于判断热点内容为何出圈、能否借势、如何转化为景区宣传角度。

## Problem Statement

甲方拥有前山牧场四季牧歌民俗风情园、伊吾及前山周边多个文旅资源，但缺少快速跟进互联网热点并转化为宣传内容的工具。泛新闻热点并不一定适合文旅宣传，系统需要更重视：

- 文旅热点是否与新疆、草原、避暑、研学、团建、亲子、自驾、红色教育、民族风情等主题相关；
- 热点内容背后的情绪价值和传播结构；
- 热点是否能自然导向甲方景点，而不是生硬蹭热点；
- 不同平台的热点表达方式，例如抖音重视频内容、小红书重种草笔记、微博重话题扩散。

## User Stories

### 1. 作为景区宣传运营人员，我想看到文旅相关热点，而不是泛新闻热点

Acceptance Criteria:

- 系统能按“文旅优先”筛选或排序热点。
- 系统能显示热点来源平台、关键词、热度、发布时间或更新时间。
- 系统能给出热点与甲方资源的匹配理由。

### 2. 作为运营人员，我想知道一个热点能不能蹭

Acceptance Criteria:

- 系统为每个热点输出匹配度，例如高/中/低或 0-100 分。
- 系统说明可借势角度，例如“新疆小众秘境”“草原避暑”“亲子研学”“团建目的地”。
- 对不适合强蹭的热点给出风险提示。

### 3. 作为运营人员，我想分析抖音热门视频为什么火

Acceptance Criteria:

- 系统可记录热门视频的标题、话题标签、作者、点赞/评论/收藏/分享、发布时间、视频时长等基础指标。
- 系统可从视频文案、字幕、画面和评论中提取传播要素。
- 系统可总结视频爆点，例如开头钩子、场景、情绪、人物、音乐、反差、路线、攻略价值。
- 系统能输出“可迁移到甲方景点的内容模板”。

### 4. 作为运营人员，我想把热点转化成可发布内容

Acceptance Criteria:

- 系统根据热点和甲方背景生成成稿，而不是只给写作建议。
- 生成内容应自然嵌入四季牧歌、伊吾胡杨林、幻彩湖、鸣沙山、乃楞格尔草原、红石峪等资源。
- 用户可进入编辑页继续修改、插图、导出长图。

## Functional Requirements

### FR1. 数据源扩展

系统应支持或规划以下数据源优先级。

#### 一级优先：文旅传播强相关平台

1. 抖音
   - 价值：短视频趋势强，适合发现旅行、景点、草原、露营、亲子、团建等视觉化热点。
   - 关注内容：热门视频、话题榜、搜索关键词结果、同城/区域内容、文旅达人内容。
   - 适合分析：视频爆点、镜头场景、字幕文案、评论情绪、话题标签。

2. 微博
   - 价值：热搜和话题传播快，适合发现公共话题、节假日出游、文旅事件、区域出圈新闻。
   - 关注内容：热搜榜、文旅话题、城市/新疆相关话题、节假日出行讨论。
   - 适合分析：话题扩散、舆情倾向、可否安全借势。

3. 小红书
   - 价值：目的地种草、攻略、拍照机位、避暑/露营/亲子游等强转化内容。
   - 关注内容：新疆旅游、哈密旅游、草原露营、小众旅行、人生照片等关键词。
   - 适合分析：用户出行决策、标题模板、封面风格、图文结构。

#### 二级优先：新闻与综合热点平台

4. 今日头条
   - 价值：已有基础数据源，可继续用于泛热点和文旅新闻补充。
   - 用法：应增加文旅过滤和匹配评分，避免被泛政治、社会新闻淹没。

5. 百度热搜 / 百度指数
   - 价值：可观察搜索意图变化，例如“新疆旅游”“避暑去哪”“草原露营”等。
   - 用法：适合作为中长期选题趋势辅助。

6. 知乎
   - 价值：适合攻略型、问答型内容，例如“新疆有哪些小众景点”“夏天去哪避暑”。
   - 用法：提取用户关心的问题和内容结构。

#### 三级补充：旅游垂直与位置平台

7. 携程/马蜂窝/去哪儿
   - 价值：目的地热度、攻略、景区评价、线路灵感。
   - 注意：抓取与使用需关注平台条款。

8. 高德/百度地图 POI
   - 价值：景区搜索热度、评论关键词、交通路线、周边设施。
   - 用法：辅助判断景点转化路径和用户痛点。

9. B站
   - 价值：长视频攻略、旅行 Vlog、纪录片式内容。
   - 用法：补充深度内容结构和年轻用户表达。

### FR2. 文旅热点匹配评分

系统应对每条热点计算“文旅借势匹配度”。建议评分维度：

- 主题相关性：是否涉及旅游、自然风光、草原、避暑、亲子、研学、团建、自驾、新疆、民族风情等。
- 区域相关性：是否涉及新疆、哈密、伊吾、东天山、草原、边疆、西北等。
- 情绪价值：是否具备松弛感、向往感、治愈感、反差感、人生照片、逃离城市等传播情绪。
- 可转化性：是否能自然导向四季牧歌或周边景点。
- 风险性：是否涉及负面舆情、灾害、冲突、政治敏感、社会悲剧等不适合蹭热点的内容。

输出示例：

```json
{
  "score": 86,
  "level": "高匹配",
  "reason": "该热点围绕夏季避暑和草原旅行，与前山牧场高海拔草原、民宿、演艺和亲子团建高度相关。",
  "recommended_angles": ["草原避暑", "亲子户外", "企业团建", "新疆小众目的地"],
  "risk": "低"
}
```

### FR3. 抖音热门视频分析方法

抖音不应只抓标题和链接，还应尽量分析视频为什么火。建议采用“元数据 + 文案 + 画面 + 声音 + 评论”的多层分析。

#### 3.1 元数据分析

可采集字段：

- 视频标题/描述
- 话题标签
- 作者昵称和账号类型
- 发布时间
- 视频时长
- 点赞数
- 评论数
- 收藏数
- 分享数
- 音乐/原声信息
- 视频链接

可计算指标：

- 互动率：点赞、评论、收藏、分享相对播放量或账号粉丝量的比例
- 分享倾向：分享数/点赞数，用于判断是否具备传播价值
- 收藏倾向：收藏数/点赞数，用于判断是否具备攻略或决策价值
- 评论热度：评论数/点赞数，用于判断话题讨论性

如果无法获得播放量，可先用可见互动指标构造相对热度分。

#### 3.2 文案与字幕分析

可分析：

- 开头 3 秒钩子：是否有强问题、强反差、强画面承诺
- 标题模板：例如“新疆不是只有赛里木湖”“这个夏天去一次草原吧”
- 字幕关键词：避暑、草原、松弛感、自驾、亲子、民宿、星空、人生照片等
- 行动召唤：收藏、转发、艾特朋友、假期出发等

技术方法：

- 如果能获取视频字幕，直接做关键词和结构分析。
- 如果没有字幕，可下载音频后用 ASR，例如 Whisper，转写语音。
- 对转写文本做关键词抽取、情绪分类、内容结构识别。

#### 3.3 画面分析

可分析：

- 场景：草原、雪山、湖泊、胡杨、峡谷、毡房、星空、骑马、演艺、团建活动等。
- 镜头节奏：航拍、转场、第一视角、人物出镜、打卡点展示。
- 视觉情绪：壮阔、治愈、热闹、浪漫、亲子、冒险、民族风。
- 可复用拍摄模板：开场大景、人物奔跑、毡房晚会、草原骑马、日落湖面等。

技术方法：

- 抽取关键帧，例如每 1-2 秒或镜头切换时取帧。
- 用图像识别/多模态模型给关键帧打标签。
- 对画面标签与甲方资源标签做匹配，例如“草原+星空+民宿”匹配四季牧歌。

#### 3.4 评论分析

可分析：

- 用户是否表达“想去”“在哪里”“怎么去”“多少钱”“适合带孩子吗”等转化意图。
- 用户关心的问题：交通、住宿、价格、季节、拍照点、安全性。
- 情绪倾向：向往、惊讶、质疑、吐槽、担忧。

输出可用于改写文案的 FAQ 或评论区引导语。

#### 3.5 热门视频到甲方景区的迁移模板

系统应把视频分析结果转化为可执行建议：

```text
热门视频爆点：夏季高温下的草原避暑反差
可迁移资源：前山牧场平均海拔约 2400 米，气候凉爽，周边有乃楞格尔草原、鸣沙山、幻彩湖
建议标题：别再只盯着热门景区了，新疆这个草原避暑地真的藏不住了
建议视频结构：城市高温开场 → 草原大景转场 → 住进毡房/木屋 → 晚上看演出和星空 → 周边顺路打卡幻彩湖/鸣沙山
```

### FR4. 甲方资源知识库

系统应维护一份甲方背景知识库，用于热点匹配和文案生成。初始内容来自：

- 《伊吾及前山附近景点介绍.docx》
- 《四季牧歌简介.docx》

知识库应至少包含：

- 景点名称
- 地理位置
- 核心卖点
- 适合人群
- 适合季节
- 可关联热点类型
- 禁止编造/需核实信息

### FR5. 输出内容形态

对每个可借势热点，系统可以输出：

- 热点摘要
- 借势理由
- 匹配景点
- 推荐标题
- 小红书/公众号/微博/抖音口播文案
- 视频脚本或分镜建议
- 图片素材建议
- 风险提示

## Non-Functional Requirements

1. 合规性：数据来源应优先使用官方接口、公开榜单、授权工具或人工导入；不应绕过平台风控或违反平台条款。
2. 可解释性：热点匹配必须给出原因，不能只给分数。
3. 安全性：对负面、敏感、灾害、事故类热点应默认不建议商业借势。
4. 可扩展性：数据源应设计为插件化，便于后续新增微博、抖音、小红书等来源。
5. 可编辑性：生成内容仍需进入现有可视化编辑器，由用户最终把关。

## Success Criteria

- 用户能看到文旅优先的热点列表，而不是纯泛新闻热榜。
- 每条热点都有“适不适合蹭”的判断和原因。
- 系统能至少支持今日头条 + 微博 + 抖音/抖音数据导入方案中的一种初版路径。
- 系统能基于甲方背景自动推荐关联景点和宣传角度。
- 对抖音热门视频，系统能输出爆点拆解和可迁移脚本建议。
- 生成文案能自然宣传四季牧歌、伊吾胡杨林、幻彩湖、鸣沙山、乃楞格尔草原等资源。

## Constraints & Assumptions

- 抖音和微博的正式接口权限有限，初期可考虑人工导入、第三方数据工具导出、公开榜单采集或半自动采集方案。
- 不保存或硬编码任何平台账号密码、Cookie、Token。
- 视频下载、字幕提取、评论采集等能力必须评估平台条款和技术可行性。
- 甲方资料以当前附件为初始知识库，后续价格、开放时间、活动频次等信息需要人工核实。
- 数据库可使用用户提供的 PostgreSQL 17.9 外部数据库，但连接凭证应放在本地 `.env.local` 或部署环境变量中，不进入仓库。

## Out of Scope

- 本阶段不实现完整用户登录/历史记录功能。
- 本阶段不直接开发抖音爬虫。
- 本阶段不绕过抖音/微博登录、加密、风控或反爬机制。
- 本阶段不承诺获得抖音播放量等非公开指标。
- 本阶段不替代人工审核，尤其是热点风险和景区事实核实。

## Dependencies

- 甲方背景知识库：`.claude/context/client-tourism-background.md`
- 后续数据库：PostgreSQL 17.9 外部数据库
- 可能的数据工具：微博热搜/话题榜、抖音热点榜、巨量算数、巨量星图、蝉妈妈、飞瓜数据、新榜、第三方榜单导出等
- 现有编辑器：HotStream 可视化块编辑器与长图导出能力

## Recommended First Implementation Slice

建议先做一个低风险 MVP：

1. 保留今日头条数据源。
2. 新增“文旅关键词过滤 + 匹配评分”。
3. 支持手动录入/导入抖音或微博热点链接与基础指标。
4. 建立甲方景点知识库。
5. 生成“热点借势分析 + 文案成稿 + 抖音脚本建议”。

这样可以先验证产品方向，再决定是否投入更复杂的自动化抖音/微博数据采集。
